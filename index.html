The changing web

  • The advent of AJAX marked a web shift
  • Sites became more responsive
  • Front-ends relied less on backend servers
  • Application logic moved into the front-end
  • With rendering, logic and computation distributed to user machines, the backend could be more easily scaled

Front-end Client Computing

  • If you live long enough you see cycles repeat
  • This so-called modern approach has similarities with native desktop client software which communicates with remote servers to pull or post data

Handling front-end complexity

  • Developers find they need ways to handle the increased code in the front-end
  • So they consider how large applications are built using software libraries and frameworks
  • They begin developing open-source front-end web frameworks with names like: Backbone, Ember, Knockout and Angular

AngualrJS

  • Angular is loosely similar to many of the other frameworks that emerged to address the increasing demands of modern web development
  • However, the similarities quickly give way to an amazing array of cleanly integrated and forward thinking features
  • We'll cover many of Angular's features in this presentation

Why AngularJS?

  • Embraces the spirit and potential of HTML5
  • Offers features which are designed to work together
  • Simplifies building modern web application

How is Angular different from other frameworks?

  • A key diferenciatior is the extent to which Angular embraces HTML and modern web development idioms
  • You might think this makes Angular a large blotted framework, but as we'll see later this isn't the case

Who owns Angular?

Angular is an open source project which is supported by its founder and a dedicated team at Google

As an open source project many of Angular's features are the result of contributions from the open source community

What can Angular do for you?

  • Make it easier for you to reason about and build your web application - regardless of its size and complexity
  • Applications which embrace Angulars features are inheritenly more testable
